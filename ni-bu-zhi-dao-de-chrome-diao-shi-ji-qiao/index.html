你不知道的Chrome调试技巧

· 2020-06-02 ·

今天上午花了大概一个小时阅读了你不知道的 Chrome 调试技巧这本掘金小册，收获颇多，并且也认识到，系统的学习是多么的重要，所以在这里记录下看完这本小册，我认为可以提高效率的技巧。强推这本小册，只需要0.01元。

Dev Tools

  • 元素面板
  • 控制台面板
  • 源代码面板
  • 网络面板
  • 性能面板
  • 内存面板
  • 应用面板
  • 安全面板

常用快捷键

  • F12 打开 Dev Tools。
  • ctrl + shift + D 快速切换 Dev Tools 的位置（top or bottom）。
  • ctrl + shift + M 快速切换 手机模式还是PC模式。
  • ctrl + P 启动命令模式

使用Command

❝

Dev Tools 内置了非常多的 Command，使用 ctrl + P 开启命令模式

❞
  • > Capture full size screenshot 截取全屏。
  • > Capture node screenshot 截取某个节点。
  • > layout 切换 Dev Tools 的排版。
  • > theme 切换 Dev Tools 的主题，有黑色和白色两种。
  • ! xxx 运行对应的代码片段

代码片段的使用

❝

在编辑器中我们可以使用代码片段来提高我们的开发速度，同样的Dev Tools有相同的功能可以提高你的调试速度。

❞

按照上图的步骤新建你的代码片段，你可以选择右击run。 也可以使用上面的命令! xxx 运行对应的代码片段。

变量的使用

  • $ 和jQuery中一个意思都是选择器
  • $0 当前选中的dom元素 $1 上一个选择的 $2 以此类推。
  • $$ 相当于document.QuerySelectorAll，但是返回值是一个Array 而不是Node list。
  • $_ 上一次运行的结果。
  • $i 相当于npm i，用来下载npm包，需要先下载Console Importer 插件。

console编写异步函数

❝

console 本身的运行环境就在async中，所以可以直接使用await

❞
const res = await fetch('xxx');
const json = await res.json();

巧用console，拥有更好的阅读体验

  • console.table() 将类数组的变量以table的形式打印出来。
  • console.log({ name, age }) 打印一个对象，更好的知道每个值对应的是什么意思。
  • console.dir() 打印dom节点
  • console.assert(Boolean, ...) 根据第一个参数来判断是否打印后面的值，减少判断语句。

Dev Tools 方法的使用

  • copy 可以将内容copy到剪切板，我们经常打印一个对象的时候，没办法复制，则可以使用这个方法。
  • queryObjects 查询对象。
  • monitor 监听function
  • monitorEvents 监听事件

元素面板

快捷键

  • h 快速隐藏对应的元素
  • ctrl + c 快速复制元素

右击 expand recursively 快速展开所有节点

DOM 断点

  • 选择 subtree modifications :监听任何它内部的节点被 移除 或者 添加的事件
  • 选择 attribute modifications :监听任何当前选中的节点被 添加，移除 或者 被修改值的事件
  • 选择 node removal :监听被选中的元素被 移除 的事件

workspace 技巧

一句话总结，就是把文件拖到Source面板，然后修改代码，既可以直接看效果，也可以将本地的文件修改。
